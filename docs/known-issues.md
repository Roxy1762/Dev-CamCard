# 当前关键问题清单（Known Issues）

> 本清单聚焦当前最影响继续推进的主链问题，优先级默认高于新增机制开发。

## 1) 日程槽合法性
- 问题：日程槽在“可安排对象 / 触发时机 / 客户端入口”上仍有一致性风险。
- 影响：规则理解与实战行为可能不一致，易出现误操作与争议。
- 优先级：P0。

## 2) 场馆耐久公开与显示
- 问题：场馆耐久虽已接入公开视图，但仍需持续防止字段回退或局部显示错位。
- 影响：玩家对场馆可击毁性判断失真，影响攻击决策。
- 优先级：P0。

## 3) 攻击分配 UI 仍偏 MVP
- 问题：客户端已补齐“攻击场馆”与 guard 提示，但仍缺少细粒度多段攻击分配器。
- 影响：高阶玩家无法一次性精确拆分多段伤害，只能通过多次快捷按钮完成。
- 优先级：P1。

## 4) 可复现回放尚未闭环
- 问题：seeded RNG 基础已落地，但客户端回放仍是事件日志视图，尚未真正按事件流重建状态。
- 影响：调试、验 bug、赛事仲裁仍缺少“逐帧可验证”的播放器。
- 优先级：P0。

## 5) schema / 类型漂移回归风险
- 问题：effect schema 已收紧，但协议层与旧文档仍可能遗留枚举/字段漂移（例如 rarity 历史值兼容）。
- 影响：后续内容扩写时，若未同步协议与文档，仍可能出现静态定义落后于运行数据。
- 优先级：P1。

## 6) 市场供给仍需观测调优
- 问题：rarity copies 已落地，但实际牌堆厚度、轮转速度与购买密度还未经过足够对局验证。
- 影响：虽然结构正确，但平衡性与构筑多样性仍可能偏移。
- 优先级：P1。

## 7) starter / fixed supplies / pressure 平衡问题
- 问题：三者尚未完成结构性重做，节奏与反制链条不稳定。
- 影响：早期决策空间与中后期转化质量不足。
- 优先级：P1。

---

## 处理原则

1. 先修规则正确性与可复现性，再扩展复杂机制。
2. 每个问题修复后更新 `docs/current-capabilities.md` 与 `docs/roadmap-next.md`。
3. 保持“文档描述、协议定义、引擎行为、客户端入口”四者一致。
