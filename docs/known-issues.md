# 当前关键问题清单（Known Issues）

> 本清单聚焦当前最影响继续推进的主链问题，优先级默认高于新增机制开发。

## 1) 日程槽合法性
- 问题：日程槽在“可安排对象 / 触发时机 / 客户端入口”上仍有一致性风险。
- 影响：规则理解与实战行为可能不一致，易出现误操作与争议。
- 优先级：P0。

## 2) 场馆耐久公开与显示
- 问题：场馆耐久虽已接入公开视图，但仍需持续防止字段回退或局部显示错位。
- 影响：玩家对场馆可击毁性判断失真，影响攻击决策。
- 优先级：P0。

## 3) 攻击场馆 UI 与 guard 场景完整性
- 问题：客户端对“攻击场馆”与 guard 约束的可操作路径仍不完整。
- 影响：引擎可执行但玩家不可达，或提示不一致。
- 优先级：P0。

## 4) RNG 非确定性
- 问题：随机流程尚未形成端到端 deterministic 策略。
- 影响：复盘不可稳定复现，调试与竞赛公平性受影响。
- 优先级：P0。

## 5) schema 过松
- 问题：effect 字段约束仍偏宽，存在“内容定义与引擎读取漂移”风险。
- 影响：线上才暴露的规则异常概率升高。
- 优先级：P0。

## 6) 市场供给结构过薄
- 问题：市场供给仍偏向 singleton，缺乏 rarity copies 的稳定支撑。
- 影响：抽样波动大，平衡与可构筑性不足。
- 优先级：P1。

## 7) starter / fixed supplies / pressure 平衡问题
- 问题：三者尚未完成结构性重做，节奏与反制链条不稳定。
- 影响：早期决策空间与中后期转化质量不足。
- 优先级：P1。

---

## 处理原则

1. 先修规则正确性与可复现性，再扩展复杂机制。
2. 每个问题修复后更新 `docs/current-capabilities.md` 与 `docs/roadmap-next.md`。
3. 保持“文档描述、协议定义、引擎行为、客户端入口”四者一致。
